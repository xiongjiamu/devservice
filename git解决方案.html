我们的Git解决方案

某大型世界500强公司有大量的软件开发团队和项目，但各个部门之间代码不可见，导致大量的代码重用，据统计大型软件公司的代码重复平均为~30%， 怎样解决代码的复用成为一个问题。

CSDN Git 解决方案借鉴开源社区的模式，在该公司实施内部开源平台，将好的项目共享出来，方便其他部门访问，并使用社区的机制，激励大家对内部开源出来的项目做贡献，不仅大大提升了代码复用率而且活跃了开发团队文化。

图片

不同研发中心的代码同步

大型公司通常有多地研发中心，通过专线来链接各个研发中心，如果代码版本控制服务器设置在一个研发中心，其他中心都需要通过专线才能访问到代码，大量的访问会导致网络拥塞和速度变慢，尤其不利于持续集成场景。

CSDN Git解决方案能够上线多研发中心的代码实时同步，每个研发中心人员可以就近访问代码仓库，通过在某银行测试，代码的获取速度提高了10倍以上。

图片

代码在线Review机制

代码Review往往是解决质量为题的第一道关口，CSDN Git解决方案提供在线的代码Review机制，可以方便的进行代码评审，包括行内评注、投票等功能。

某大型软件公司使用了合并请求来Review代码，据统计代码缺陷率降低了20%以上。

图片

不同环境中的代码同步

某银行的研发中心区分开发环境和生产环境，开发环境和生产环境严格隔离，如果将开发环境的代码部署到生产环境需要先用FTP传到中转服务器上，让后再转到生产环境部署，对于持续部署效率很低。

我们利用分布式Git方案在生产和开发环境部署CODE服务，实现代码同步并严格限制代码读写，保证开发环境代码更新后实时同步到生产环境。
